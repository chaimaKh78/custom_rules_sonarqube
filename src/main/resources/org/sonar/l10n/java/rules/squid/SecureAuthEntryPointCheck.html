Secure Authentication Entry Point Check

This rule verifies that the AuthenticationEntryPoint implementation properly handles unauthorized access.

Why is this important?

Unauthorized access attempts should be handled consistently and securely, ensuring that the correct HTTP status code is returned and that errors are logged appropriately.

Criteria for Security

  • The sendError method must use HTTP status code 401 for unauthorized errors.
  • Exceptions occurring during this process should be logged using a logging framework.

How to fix issues raised by this rule

  • Ensure that unauthorized access is handled by returning the HttpServletResponse.SC_UNAUTHORIZED status.
  • Check that exceptions are logged using a logger, especially in the context of security events.

Examples

Here are examples demonstrating the appropriate and inappropriate usage of the commence method:

    // Bad Example: Not using HTTP status 401
    response.sendError(403); // Forbidden

    // Bad Example: Not logging exceptions
    response.sendError(HttpServletResponse.SC_UNAUTHORIZED);

    // Good Example: Properly handling unauthorized access with logging
    response.sendError(HttpServletResponse.SC_UNAUTHORIZED);
    logger.error("Unauthorized access attempt detected.");
