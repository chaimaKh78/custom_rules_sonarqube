Password Encoder Rule

This rule checks for the secure usage of password encoders and hashing methods in your codebase.

Why is this important?

Using weak password encoders or hashing methods can lead to security vulnerabilities. It's crucial to adopt strong and modern practices to protect user credentials.

Criteria for Security

  • Only strong password encoders such as BCryptPasswordEncoder or Argon2PasswordEncoder should be used.
  • Avoid using weak hashing methods like MD5 and SHA1.
  • In critical authentication methods, ensure the use of secure password encoders.

How to fix issues raised by this rule

  • Replace weak password encoders with secure options, such as BCryptPasswordEncoder.
  • Avoid using weak hashing algorithms. Use strong alternatives like SHA-256 or stronger.
  • Ensure that critical authentication methods utilize secure password encoders.

Examples

Here are examples demonstrating the appropriate and inappropriate usage of password encoders:

    // Bad Example: Using a weak encoder
    PasswordEncoder encoder = new NoOpPasswordEncoder();

    // Bad Example: Using weak hashing method
    String hashedPassword = MD5(password);

    // Good Example: Using a secure encoder
    PasswordEncoder encoder = new BCryptPasswordEncoder();
    String hashedPassword = encoder.encode(password);
