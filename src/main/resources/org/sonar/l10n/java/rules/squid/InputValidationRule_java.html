Input Validation Rule

This rule ensures that input parameters in your methods are validated using @Valid or equivalent validation annotations.

Why is this important?

Proper input validation is crucial for application security. Failing to validate input can lead to various security vulnerabilities, including injection attacks and data corruption.

How to fix issues raised by this rule

  • Annotate your input parameters with @Valid or similar annotations to ensure they are validated before processing.
  • Use appropriate validation frameworks to enforce constraints on the input data.

Examples

Here are examples demonstrating proper input validation:

    // Bad Example: Missing validation
    @GetMapping("/user")
    public User getUser(@RequestParam String id) {
        // User not validated
        return userService.findById(id);
    }

    // Good Example: Using @Valid for validation
    @PostMapping("/user")
    public User createUser(@RequestBody @Valid User user) {
        return userService.save(user);
    }
