Inefficient Database Calls

This rule aims to prevent inefficient database operations by avoiding saving entities inside loops.

Why is this important?

Executing save operations inside loops can lead to significant performance issues, as each operation may result in a separate database transaction. Using batch updates instead can drastically improve performance.

How to fix issues raised by this rule

  • Refactor your code to accumulate entities and save them in a single batch operation after the loop.
  • Consider using methods such as saveAll instead of individual save calls within loops.

Examples

Here are examples demonstrating how to avoid inefficient database calls:

    // Bad Example: Saving inside a loop
    for (Entity entity : entities) {
        repository.save(entity); // Inefficient
    }

    // Good Example: Using saveAll
    repository.saveAll(entities); // Efficient
