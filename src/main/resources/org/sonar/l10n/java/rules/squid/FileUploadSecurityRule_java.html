File Upload Security Validation

This rule ensures that uploaded files are properly validated for their type, size, and content, and that they are scanned for malware before being saved. It promotes secure handling of file uploads to prevent potential vulnerabilities.

Why this rule matters

File uploads can pose significant security risks if not handled correctly. Validating uploaded files helps mitigate risks such as malicious file uploads, and scanning for malware ensures that harmful files are not processed by the application.

Compliant Code Example


public void uploadFile(File file) {
    if (isValidExcelFile(file) && validateFileSize(file)) {
        // Scanning for malware before saving
        scanForMalware(file);
        save(file);
    }
}

Non-Compliant Code Example


public void uploadFile(File file) {
    // No validation or scanning performed before saving
    save(file);
}

Additional Considerations

  • Always validate file uploads to ensure they meet your application's requirements.
  • Consider using a library or service for malware scanning to ensure comprehensive protection.
  • Log validation failures to monitor and respond to potential security threats.
